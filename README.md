# ESP32 SDM530 S0 Pulse Power Meter

ESPHome firmware for an ESP32 that reads the S0 pulse output of an **Eastron SDM530** three-phase energy meter and exposes real-time power and persistent energy sensors to **Home Assistant**.

---

## Requirements

- ESPHome 2026.3 or newer
- Home Assistant with the ESPHome integration
- An ESP32 dev board (`esp32dev` pinout)
- Eastron SDM530 energy meter
- Two resistors: **10 kΩ** (R1) and **20 kΩ** (R2), ¼ W or better

---

## Hardware

| Component | Value | Purpose |
|---|---|---|
| ESP32 dev board | any `esp32dev` pinout | Main controller |
| Eastron SDM530 | — | Three-phase energy meter |
| R1 | 10 kΩ, ¼ W | Series current limiter on signal line |
| R2 | 20 kΩ, ¼ W | Pull-down; forms voltage divider with R1 |

---

## Wiring

The SDM530 S0 output (terminals 8 & 9) is a **passive NPN open-collector transistor** — it has no internal power source. The transistor closes briefly once per energy pulse, connecting Pin 9 to Pin 8.

Both resistors go on the **Pin 8 (output) side only**, so Pin 9 receives the full unloaded 5 V required by the Eastron spec.

```
ESP32 5V (VIN) ──────────────────── SDM530 Pin 9  (S0 +)
                                          │
                             [NPN transistor — inside SDM530]
                                          │
                                 SDM530 Pin 8  (S0 −)
                                          │
                                 R1 10 kΩ  (series)
                                          │
ESP32 GPIO18 ──────────────────────────── ┤
                                          │
                                 R2 20 kΩ  (pull-down)
                                          │
ESP32 GND ───────────────────────────── GND
```

### Why the resistors are needed

The ESP32 GPIO is rated for **3.3 V maximum**. The S0 supply is 5 V. R1 and R2 form a voltage divider that drops the pulse to a safe level:

$$V_{GPIO} = 5\text{ V} \times \frac{20\text{ k}\Omega}{10\text{ k}\Omega + 20\text{ k}\Omega} = \textbf{3.33 V}$$

| State | Transistor | GPIO18 | Logic |
|---|---|---|---|
| Idle | Open | 0 V (R2 pull-down) | LOW |
| Pulse | Closed | 3.33 V (divider) | HIGH |

Current on pulse: `5 V / 30 kΩ = 0.17 mA` — well under the meter's 27 mA limit and safe for the GPIO.

## Installation

### 1 — Copy the files

Copy the following file into your ESPHome config directory (the folder where `secrets.yaml` lives):

```
esp32-sdm530.yaml
```


### 2 — Set your credentials

Open `esp32-sdm530.yaml` and fill in the three device-specific secrets generated by the ESPHome dashboard. Each has two options — pick one and comment out the other.

**Option A — paste directly into the file** (keep this file out of git):
```yaml
key: "your_api_key_here"
password: "your_ota_password_here"
```

**Option B — store in `secrets.yaml`** (recommended for git repos):

Add to `secrets.yaml`:
```yaml
esphome_api_key: "your_api_key_here"
esphome_ota_password: "your_ota_password_here"
esphome_ap_password: "your_ap_password_here"
```

See `secrets.yaml.example` for the full template.

> **Where to find these values:** In the ESPHome dashboard, open the device → **Edit**. The API key, OTA password, and fallback AP password are auto-generated when the device is first added.

### 3 — Set the impulse constant

At the top of `esp32-sdm530.yaml`, set `pulses_per_kwh` to match your meter's label (printed on the front). Common values are 800, 1000, and 2000.

```yaml
substitutions:
  pulses_per_kwh: "800"   # ← check your meter label
```

### 4 — Flash the device

In the ESPHome dashboard, click **Install** on the `esp32-sdm530` device. On first flash, use a USB cable. Subsequent updates are OTA.

### 5 — Add to Home Assistant

If not auto-discovered, go to **Settings → Devices & Services → Add Integration → ESPHome** and enter the device's IP address.

## Configuration

### LED pin

GPIO2 is correct for most ESP32 DevKit boards. If the LED does not blink on pulses after flashing:

1. Flash `esp32-led-finder.yaml` to the same board.
2. On boot it cycles through GPIO2 / 4 / 5 / 16 / 17, each with a distinct blink pattern — count the blinks to identify your pin.
3. Update `pin: GPIO2` in the `output:` block of `esp32-sdm530.yaml` with the correct pin.

### Noise / false pulses

If you get phantom power readings, check wiring first. The firmware already applies:

| Setting | Value | Effect |
|---|---|---|
| `internal_filter` | 250 ms | Ignores any two rising edges closer than 250 ms |
| `internal_filter_mode` | EDGE | Filters on time between edges — never rejects a valid pulse by width |
| `pulldown: true` | ~45 kΩ internal | In parallel with R2 → ~14 kΩ effective; hardens pin against capacitive noise |

At 800 imp/kWh and 15 kW max load, the minimum real pulse interval is 300 ms — the 250 ms filter rejects noise while accepting pulses up to ~18 kW.

## Sensors & Entities

All entities appear under the **ESP32 SDM530** device in Home Assistant.

### Always visible

| Entity | Unit | Description |
|---|---|---|
| SDM530 Power | W | Instantaneous power — updates on every pulse |
| SDM530 Energy Total | kWh | Cumulative energy — persists across reboots (NVS flash, saved every 60 s) |
| SDM530 Daily Energy | kWh | Energy since midnight — resets at midnight; also persists across reboots |

### Hidden by default

These are disabled by default to avoid cluttering the UI. Enable them in HA under **Settings → Devices & Services → ESPHome → your device** when needed.

| Entity | Type | Purpose |
|---|---|---|
| SDM530 Reset Energy Total | Button | Zeroes the Energy Total counter — use when replacing the meter |
| SDM530 Set Energy Total | Number (kWh) | Type the kWh reading from the physical meter display to sync them |

## Energy Counter Management

### Syncing to the physical meter

When commissioning on an existing installation, the ESP starts at 0 kWh while the meter shows (for example) 12 345 kWh. To sync:

1. In HA, enable the **SDM530 Set Energy Total** entity.
2. Enter the kWh value shown on the meter display.
3. The ESP converts it to pulses and restores from that point. The NVS value is updated immediately.

### Resetting to zero

Press the **SDM530 Reset Energy Total** button. This zeroes both the live counter and the NVS saved value instantly.

### How persistence works

The raw pulse count is saved to the ESP32's non-volatile flash (NVS) every 60 seconds. On every boot, the saved count is restored before the pulse meter begins counting — so the energy total survives reboots and power cuts.

Worst-case data loss on sudden power cut: 60 seconds of pulses ≈ 0.25 kWh at 15 kW peak load.

The maximum value storable in the ESP32 uint32_t counter at 800 imp/kWh is ≈ **5,369 MWh**. Beyond this, the ESP counter wraps around, but Home Assistant's `total_increasing` state class detects the rollover and keeps the lifetime total correct in its own database.

### HA-side persistence (alternative)

`ha-energy-persistence.yaml` is an optional Home Assistant package that saves the pulse count as an `input_number` helper and restores it to the ESP when it reconnects. Use this only if you want to manage the energy total from the HA side instead of the ESP32's own NVS flash. **Do not use both at the same time.**
